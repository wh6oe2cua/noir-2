# 2. ゲームシステム・ルール

## 基本ルール
1. **ジャンルの選択**: ゲーム開始前に、プレイヤーは出題される問題のジャンル（例: 歴史、科学、IT・技術、雑学、ランダム）を選択します。
2. **難易度のランダム決定**: 各問題の開始時に、「普通」または「難しめ」の難易度がランダムに決定され、それに基づいた問題が生成されます。
3. **問題の開始**: ゲームが始まると、選択されたジャンルと難易度に基づき、ローカルの問題データバンクから「もっともらしい嘘が1箇所だけ含まれた文章」が抽出されます。
4. **間違いの指摘**: 文章はいくつかの「文節」や「単語」のパーツに分かれて表示されます。プレイヤーは、その中で「事実と異なる（嘘である）パーツ」を直接クリック・タップして選択します。
4. **ライフ（HP）**: プレイヤーのライフは最大 **3個** です。間違ったパーツを選択するとライフが1減ります。
5. **制限時間**: 1問あたりの制限時間は **60秒** です。時間がゼロになるとライフが1減り、強制的に次の問題へ進みます。
6. **クリア条件**: 全 **6問** に挑戦します。途中でライフが0になった場合は、その時点でゲームオーバーとなります。
7. **チェンジ機能**: 各問題につき **1回のみ**、表示されている問題を別の未回答の問題へチェンジ（変更）することができます。解答にどうしても自信がない場合に活用可能です。

## 問題抽出・構成フロー
本ゲームでは、あらかじめ用意されたデータバンク（PREMADE_QUESTIONS）から、プレイヤーが選択したテーマに沿った問題をランダムに抽出します。

```
 [プレイヤー] ジャンルを選択してゲーム開始 
       ↓
 [システム] 指定されたジャンル・難易度の問題をデータバンク（PREMADE_QUESTIONS）から検索
       ↓
 [システム] すでに解答済みの問題ID（seenQuestionIds）を弾いて、未解答の問題をランダムに選択
       ↓
 [システム] 以下の形式の問題データを取得
   - テーマ（選択されたジャンル）
   - 問題文（文節/単語ごとに分割可能な構造）
   - 間違いの文節のインデックス番号
   - 正しい事実（解説文）
       ↓
 [システム] 画面に文章をボタン状に並べて表示し、タイマー（60秒）をスタート
```

## スコアシステム
- **正解時の加点**: 残り時間が多ければ多いほど、高いスコアを獲得できます。
- **連続正解（コンボ）**: 連続で正解するとボーナススコアが加算されます。

## 音響・サウンド仕様
本ゲームは、Web Audio APIを使用して、外部音声ファイルを読み込むことなく、ブラウザ上でリアルタイムにBGMおよび効果音（SE）を合成・再生します。これにより、環境依存による読み込み遅延を防ぎます。

### 1. BGMの方向性
- **タイトル/コンセプト**: 「プロンプト・ノワール」
- **楽曲スタイル**: **レトロな探偵もの・フィルム・ノワール風ジャズ調アンビエント（ループ）**
- **音響構成**: 
  - 低音で響くアコースティックベース（ウッドベース）のピチカートを模した、ミステリアスなウォーキング・ベースライン（100 BPM程度、Amを基調としたスリリングな4ビート）。
  - チープで温かみのあるレトロなオルガンまたはエレピ風のマイナーなコードバッキング（三角波＋ローパスフィルタ）が背後に漂い、事件の謎解きに挑む探偵の雰囲気を醸し出します。

### 2. 効果音（SE）仕様
- **問題（文節）クリック・選択時**: 
  - 古いタイプライターを叩いたときのような「カチャッ」という乾いたメカニカルなクリック音、または木琴のようなパーカッシブなトーン。
- **正解（嘘見破り）時**: 
  - 推理が冴え渡ったことを示す、小粋なジャズ風のメジャー和音のサックス/オルガン風チャイム（「シャキーン」「パララン」）。
- **不正解（誤認）時**: 
  - 「ガーン」というジャズのミュート・トランペットを模したようなシリアスで重たい不協和音。
- **タイムアップ時**: 
  - 時間切れを告げるレトロなブザー音、または不穏な警報音。
- **チェンジ（問題変更）時**: 
  - トランプのカードを素早くシャッフルするような、または新聞をサッとめくるような軽快なノイズシンセ。
