# 4. データ構造・仕様

本ゲームは、あらかじめ用意された高品質な問題データバンク（PREMADE_QUESTIONS）の構造化されたJSONデータから、対象の問題を抽出します。

## 問題データ（JSON形式）のスキーマ定義

AIから受け取る問題オブジェクトは、以下の構造（スキーマ）とします。

```json
{
  "id": "string (問題の一意なID)",
  "theme": "string (歴史, 科学, IT・技術, 日常雑学, 詐欺対策, 論理の飛躍 などのジャンル)",
  "difficulty": "string ('normal' または 'hard' の難易度)",
  "title": "string (問題の簡単なタイトル。例: 『怪しい還付金手続き』)",
  "segments": [
    "string (文節1)",
    "string (文節2)",
    "string (文節3)",
    "..."
  ],
  "incorrect_segment_index": "number (間違いや詐欺手口、論理の飛躍が含まれている文節の配列インデックス。0から始まる数値)",
  "original_incorrect_text": "string (問題のある箇所のテキスト。検証用)",
  "correct_fact": "string (正しい事実、あるいは論理的に正しいあるべき状態・対処法。)",
  "explanation": "string (なぜその情報が詐欺手口・論理の飛躍（誤り）なのか、どうやってそれを見破るべきかの簡潔なファクトチェック解説)"
}
```

### 生成例1（詐欺対策の例 - 難易度: normal）
```json
{
  "id": "q_fraud_001",
  "theme": "詐欺対策",
  "difficulty": "normal",
  "title": "税務署を騙る還付金手続き",
  "segments": [
    "税務署から還付金の連絡が届き、",
    "「期限が迫っているため、",
    "近くの銀行のATMに行き、",
    "画面の指示に従って操作してください」と",
    "手続きを急かされた。"
  ],
  "incorrect_segment_index": 2,
  "original_incorrect_text": "近くの銀行のATMに行き、",
  "correct_fact": "ATMで還付金が支払われることは絶対にありません",
  "explanation": "公的機関が還付金の手続きのためにATMの操作を指示することは絶対にありません。ATMの操作によってお金が戻ることはなく、むしろ犯人の口座にお金を振り込ませる手口（還付金詐欺）です。"
}
```

### 生成例2（論理の飛躍の例 - 難易度: hard）
```json
{
  "id": "q_logic_001",
  "theme": "論理の飛躍",
  "difficulty": "hard",
  "title": "読書習慣と年収の相関",
  "segments": [
    "ある調査によると、",
    "高収入のビジネスパーソンは",
    "読書量が多い傾向にあるため、",
    "本をたくさん買いさえすれば",
    "誰でも年収が上がると言える。"
  ],
  "incorrect_segment_index": 3,
  "original_incorrect_text": "本をたくさん買いさえすれば",
  "correct_fact": "本を買うことと年収向上の間に直接の因果関係は証明されていません",
  "explanation": "これは「相関関係と因果関係の混同（論理の飛躍）」です。高収入者が本をよく読むという相関関係があったとしても、「本を買うだけで年収が上がる」という直接の因果関係（特に購入するだけで読まない、実践しない場合）には結びつきません。"
}
```

## 作問時に使用したGemini APIプロンプト（参考リファレンス）
本ゲームのプリメイド問題群をGeminiを用いて安全に設計・作問する際に使用した、システム指示（System Instruction）の定義です。これをもとに高品質な問題が静的データベースとしてビルドされています：

```text
あなたは、論理的思考力や情報リテラシー、防犯意識を試す非常に知的なゲームの作問者です。
ユーザーが「もっともらしい文章」の中に潜む「嘘」「詐欺の罠」「論理の飛躍（誤謬）」をファクトチェックするクイズ問題を作成してください。

プレイヤーが選択したジャンル（歴史、科学、IT・技術、日常雑学、詐欺対策、論理の飛躍）および、指定された難易度（normal、hard）に応じて、以下のルールに従って問題文を作製してください。

【難易度設定】
- normal（普通）: 比較的広く知られている誤解、典型的な詐欺手口、または少し考えれば矛盾に気づく論理の飛躍を設定します。
- hard（難しめ）: 専門的な知識を巧みに混ぜた誤認、非常に巧妙で現実的なフィッシング手法、または前提から結論までの飛躍が極めて微妙で、一見すると説得力があるように思える高度な論理展開（誤謬）を構築します。

【ルール】
1. 文章は全体で80文字〜150文字程度とし、読みやすく自然な文節（4〜7個程度）に分割してください。
2. 分割された文節のうち、必ず【1箇所だけ】、事実とは異なる「嘘」、あるいは「詐欺・不審な手口の記述」、あるいは「論理的な飛躍（誤謬）」を仕込んでください。
3. 各ジャンルにおける仕込み方の指針：
   - 「歴史/科学/IT・技術/日常雑学」: 一見もっともらしく見える誤解、日付、人名、用語のすり替え。
   - 「詐欺対策」: フィッシングメールや還付金詐欺、怪しい投資話などで使われる「ATMへ誘導する」「暗証番号・ログイン情報を要求する」「絶対に儲かると謳う」などの不審なトリガー。
   - 「論理の飛躍」: 相関関係と因果関係の混同、極端な一般化、前提と結論が繋がらない飛躍した論理展開。
4. あまりにも突飛な嘘や、一目でわかるような不自然な日本語は避け、よく読み込まないと騙されてしまうような「もっともらしさ」を意識してください。
5. 出力は指定されたJSONスキーマに完全に従ってください。
```
