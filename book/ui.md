# 3. 画面設計・UI

シンプルなデザインコンセプトに基づき、無駄な要素を省いた使いやすいUIを設計します。

## 画面構成
アプリは以下の4つの基本画面（ステート）で構成されます。また、メインタイトルのすぐ横に小さく **「サウンド切り替え（音響オン/ミュート）」ボタン** を設置し、プレイヤーがいつでも音量をコントロールできるように配慮します。

1. **タイトル画面 (Start Screen)**
   - ゲームタイトル：「プロンプト・ノワール」
   - サブタイトル：「嘘つきAIの嘘を見抜け」
   - ルール説明（制限時間60秒、ライフ3、全6問、ファクトチェックデータバンク抽出など）
   - **ジャンル選択エリア**:
     - 選択肢: `[歴史]` `[科学]` `[IT・技術]` `[日常雑学]` `[ランダム]` からプレイヤーが1つ選択。
   - 「ゲームスタート」ボタン

2. **ゲームプレイ画面 (Play Screen)**
   - **ヘッダー**:
     - 残り時間インジケーター（60秒のプログレスバー、カウントダウン数値表示）
     - ライフ表示（ハートアイコン ❤️❤️❤️、ダメージを受けると割れるアニメーション）
     - 現在の問題数（例: 第 1 問 / 6問中）
   - **メインエリア**:
     - 「もっともらしく書かれた文章」が、文節ごとに分割されたボタン（チップ）として並びます。
     - *例*: `[織田信長は]` `[1582年に]` `[本能寺の変において]` `[豊臣秀吉によって]` `[自害に追い込まれた。]`
   - **操作**:
     - プレイヤーが間違いだと思う部分をクリック（タップ）します。

3. **判定・解説画面 (Feedback Screen)**
   - プレイヤーが選択した部分が「正解（嘘を見破った）」か「不正解（本当のことを嘘と疑った）」かを瞬時に判定します。
   - **正解の場合**:
     - 選択したパーツが緑色にハイライトされ、「正解！」と表示。
     - 正しい事実の解説文（ファクトチェック）を表示します。
   - **不正解の場合**:
     - 選択したパーツが赤色になり、本来の間違いパーツが緑色で示され、「不正解...」と表示。ライフが1減少。
     - 同様に、解説文を表示します。
   - 「次の問題へ」ボタン。

4. **リザルト画面 (Result Screen)**
   - 最終的な獲得スコア、正解数を表示。
   - プレイヤーの「ファクトチェッカー階級」（例: 初心者ファクトチェッカー、プロファクトチェッカー、AIマスターなど）の評価。
   - 「もう一度プレイする」ボタン。
